Module M04 Day 5–6

Information Architecture

If users can't find it, it doesn't exist.

Learning Objectives

  • Build a sitemap that reflects user mental models, not org charts
  • Run an open card sort and interpret the resulting clusters
  • Choose between hierarchical, sequential, and matrix navigation
  • Spot ambiguous or overlapping navigation labels
  • Apply the 3-click heuristic critically, not literally

Core Concepts

Structure is invisible until it's wrong

Good IA disappears — users simply find what they need. Bad IA is the silent reason for high bounce rates and support tickets that say 'I couldn't find...'.

Card sorting, two ways

Open card sorts ask users to create and name their own categories, revealing mental models you didn't expect. Closed sorts ask users to place items into categories you've already defined, validating a structure you've drafted.

  • Open sort — for discovering structure from scratch
  • Closed sort — for validating an existing sitemap
  • Hybrid sort — a fixed set of categories plus an 'other'

1. Sitemaps — Match Mental Models, Not Org Charts

What happens here:

  • Diagram every page/screen and how they connect
  • Structure around how users think, not how the company is organized internally

Org chart (wrong)

  • "Warehouse Category 4"

User mental model (right)

  • "Birthday Gifts"

Example

A customer doesn't think "Warehouse Category 4" — they think "Birthday Gifts." The sitemap should reflect the second one.
Birthday Giftsnot Warehouse Cat. 4

👉 Output: Navigation that matches how people already think

2. Card Sorting — Discover or Validate Structure

What happens here:

  • Open sort — users create and name their own categories (discovery)
  • Closed sort — users place items into categories you've already drafted (validation)

Key rule:

  • ~15 participants is a common target for stable open-sort clusters

Open sort

  • Users create categories
  • Discovery

Closed sort

  • Users place into yours
  • Validation

Example

An open card sort on a furniture site revealed users grouped by "room" (Bedroom, Living Room) rather than by "material," which the original nav used.
Grouped by roomnot material

👉 Output: A structure backed by how real users group things, not a guess

3. Navigation Patterns — Choose Deliberately

What happens here:

  • Hierarchical — nested like folders (Settings > Privacy > Blocked users)
  • Sequential — step-by-step (checkout flow)
  • Matrix — multiple filters at once (catalog by size + color)

Hierarchical

  • Nested folders

Sequential

  • Step-by-step

Matrix

  • Multi-filter

Example

Forcing a product catalog into hierarchical-only navigation makes "filter by size AND color" impossible — matrix navigation is required.
Filter by size + color

👉 Output: A nav pattern that matches the actual shape of the content

4. Labeling — Kill Ambiguity

What happens here:

  • Audit every nav label for words that could mean two things
  • Test labels with users, not just internal agreement

Ambiguous

  • "Resources"

Clear

  • "Blog"
  • "Downloads"

Example

"Resources" could mean blog posts, downloads, or support — pick one or split it.
Pick one or split it

👉 Output: Labels nobody has to guess about

The Structure Loop

IA gets revisited constantly: new content categories force a sitemap update, and a failed usability test on navigation sends you back to card sorting.

Simple Real-World Flow

  1. 1List all current navigation items (Sitemap audit)
  2. 2Run an open card sort with real users (Card Sorting)
  3. 3Compare their groupings to your existing nav
  4. 4Pick the navigation pattern that fits the content shape (Navigation Patterns)
  5. 5Stress-test every label for ambiguity (Labeling)

Key Takeaways

  • Good IA disappears — bad IA shows up as support tickets
  • Card sorting tells you the user's mental model, not your own
  • The right navigation pattern depends on the content's actual shape
  • Click count matters less than whether each click builds confidence

Quick Concept Check

See It In Practice

Information architecture is the practice of structuring, organizing, and labeling content so it matches how users actually look for things.
An online store organized products by internal warehouse category ('SKU Type 4'). A card sort revealed customers thought in terms of occasions ('Birthday', 'Office') — the nav was rebuilt around that instead.
Take 15 menu items from a navigation you use daily. Without looking at the original structure, group them into categories that make sense to you.

Common Questions

Not as a hard law — usability research has found click count matters less than whether each click confidently moves the user closer to their goal. A confusing 2-click path beats a clear 4-click one.

Today's Exercise

Open Card Sort

Write 20 content items from a familiar app on individual cards (physical or digital). Have a partner group and label them without your input, then compare to the live navigation.

Materials

  • 20 index cards or a digital sort tool
  • A partner

Time estimate

35 minutes

Deliverable

Photo or screenshot of the resulting groups + category labels.
