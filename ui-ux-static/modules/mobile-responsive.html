Module M09 Day 11–12

Mobile & Responsive

One design, every screen, no exceptions.

Learning Objectives

  • Apply minimum touch-target sizing for accessible tap interactions
  • Design a layout that reflows correctly across three standard breakpoints
  • Choose between adaptive and fluid responsive strategies for a given screen
  • Identify thumb-reach zones on common mobile device sizes
  • Adjust typography and spacing scale appropriately for small screens

Core Concepts

Touch is not a smaller mouse click

Fingers are imprecise and screens are held at odd angles. Interfaces designed for precise cursor clicks routinely fail the moment they're tapped on a moving train.

Breakpoints as design decisions, not afterthoughts

Each breakpoint should be a deliberate layout decision, not just 'shrink everything until it fits'.

  • Mobile (<768px) — single column, bottom-anchored primary actions
  • Tablet (768–1024px) — 2-column grids, collapsible side panels
  • Desktop (1024px+) — multi-column, persistent sidebars

3. Adaptive vs. Fluid — Two Strategies

What happens here:

  • Adaptive — snaps between fixed configurations at set breakpoints
  • Fluid — resizes continuously using relative units

The Responsive Loop

Responsive isn't "design once, resize automatically" — a breakpoint decision often forces a content priority decision, and mobile-first constraints frequently improve the desktop layout too.

Simple Real-World Flow

  1. 1Take a 4-column desktop card grid
  2. 2Decide what collapses, stacks, or gets cut at tablet (Breakpoints)
  3. 3Redesign for a 390px mobile viewport (Mobile-first decisions)
  4. 4Verify every tap target meets minimum size (Touch Targets)
  5. 5Check primary actions sit in thumb-reach zones

Key Takeaways

  • Touch needs bigger, more forgiving targets than a cursor
  • Breakpoints should be content decisions, not just CSS math
  • Mobile-first forces prioritization that improves every screen size
  • Responsive isn't the same as accessible — they're related but separate checks
