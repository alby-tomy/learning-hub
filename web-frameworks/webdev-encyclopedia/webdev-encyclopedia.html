HTTP (HyperText Transfer Protocol) is the language browsers and servers speak. Every resource — HTML file, image, CSS file — is fetched with an HTTP request.

MODULE 02

The Trinity: HTML + CSS + JS

Understanding what each language does and how they work together

Think of a webpage as a house. HTML is the structure (walls, rooms, doors). CSS is the decoration (paint, furniture, lighting). JavaScript is the electricity and plumbing — it makes everything interactive and dynamic.

📄
HTML — Structure
Defines what's on the page: headings, paragraphs, images, forms, links. It's the skeleton.
🎨
CSS — Style
Controls how everything looks: colors, fonts, sizes, layout, spacing, animations.
⚡
JavaScript — Behavior
Makes things happen: responds to clicks, updates content, talks to APIs, creates interactivity.

How Files Link Together

index.html — the entry point 
<!DOCTYPE html>
<html lang="en">
<head>
  <!-- Link CSS file -->
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <h1 id="title">Hello World</h1>
  <button id="btn">Click me</button>

  <!-- Link JS file — placed at the bottom -->
  <script src="script.js"></script>
</body>
</html>
styles.css — applies visual rules 
h1 {
  color: #264de4;
  font-size: 2rem;
}

#btn {
  background: #e34c26;
  color: white;
  padding: 10px 20px;
  border: none;
  cursor: pointer;
}
script.js — adds behaviour 
const btn = document.getElementById('btn');
const title = document.getElementById('title');

btn.addEventListener('click', () => {
  title.textContent = 'You clicked the button!';
  title.style.color = '#f7df1e';
});

Semantic HTML

Use the right element — meaning matters for SEO and accessibility

Semantic HTML uses elements that describe their meaning — not just how they look. Instead of nesting everything in <div>s, use elements that tell the browser (and screen readers) what each part of the page actually is.

Semantic Page Layout
<body>
  <header>          <!-- Site header/navigation -->
    <nav>
      <ul>
        <li><a href="/">Home</a></li>
      </ul>
    </nav>
  </header>

  <main>             <!-- Primary page content -->
    <article>       <!-- Self-contained content (blog post, card) -->
      <h1>Post Title</h1>
      <time datetime="2024-01-15">January 15, 2024</time>
      <p>Content...</p>
    </article>

    <section>        <!-- Thematic grouping of related content -->
      <h2>Related Posts</h2>
    </section>
  </main>

  <aside>           <!-- Sidebar, related content -->
    <h3>Popular Tags</h3>
  </aside>

  <footer>          <!-- Site footer -->
    <p>&copy; 2024 My Site</p>
  </footer>
</body>
💡

MODULE 07

CSS — Selectors & the Cascade

How CSS finds elements and decides which styles win

CSS works in two parts: selectors (which elements to style) and declarations (what styles to apply). The cascade is the algorithm that decides which rule wins when multiple rules target the same element.

Selector Types
/* ── Element selector — targets all <p> tags ── */
p { color: navy; }

/* ── Class selector — targets class="card" ── */
.card { background: white; }

/* ── ID selector — targets id="hero" ── */
#hero { height: 100vh; }

/* ── Attribute selector ── */
input[type="text"] { border: 1px solid gray; }

/* ── Descendant: p inside .container ── */
.container p { margin: 0; }

/* ── Child: direct children only ── */
ul > li { list-style: none; }

/* ── Adjacent sibling ── */
h2 + p { margin-top: 0; }

/* ── General sibling ── */
h2 ~ p { color: gray; }

/* ── Group multiple selectors ── */
h1, h2, h3 { font-family: sans-serif; }

/* ── Universal selector ── */
* { box-sizing: border-box; }

Specificity — Which Rule Wins?

When multiple CSS rules target the same element, the one with the highest specificity wins. Think of it as a score.

SelectorSpecificity ScoreExample
Inline style1,0,0,0 (highest)style="color:red"
ID0,1,0,0#hero
Class / attribute / pseudo-class0,0,1,0.card, :hover
Element / pseudo-element0,0,0,1p, ::before
Universal *0,0,0,0 (lowest)*
⚠️

Avoid !important — it overrides everything and makes debugging a nightmare. Use it only as an absolute last resort.

The Box Model

Every element is a box — content, padding, border, margin

Every HTML element is rendered as a rectangular box. The box model describes how the browser calculates the size and spacing of that box.

MARGIN (space outside)
BORDER
PADDING (space inside)
CONTENT
width × height
Box Model Properties
.box {
  /* Content size */
  width: 300px;
  height: 200px;

  /* Padding — inside the border */
  padding: 16px;              /* all sides */
  padding: 10px 20px;        /* top/bottom left/right */
  padding: 5px 10px 15px 20px; /* top right bottom left */

  /* Border */
  border: 2px solid #333;   /* width style color */
  border-radius: 8px;        /* round corners */

  /* Margin — outside the border */
  margin: 20px;
  margin: 0 auto;            /* center horizontally */

  /* CRITICAL: include border+padding in width calculation */
  box-sizing: border-box;
}
💡

Always put *, *::before, *::after { box-sizing: border-box; } at the top of your CSS. It makes sizing predictable — padding and border no longer add to the element's total width.

MODULE 09

Layout: Flexbox

One-dimensional layout — rows and columns made easy

Flexbox is designed for one-dimensional layout — either a row or a column. It's perfect for navbars, card rows, centering content, and any layout along a single axis.

Flexbox Essentials
.container {
  display: flex;

  /* Main axis direction */
  flex-direction: row;          /* default: left→right */
  flex-direction: column;       /* top→bottom */
  flex-direction: row-reverse;  /* right→left */

  /* Align along MAIN axis (horizontal if row) */
  justify-content: flex-start;  /* default */
  justify-content: center;      /* center */
  justify-content: flex-end;    /* right */
  justify-content: space-between; /* spread out */
  justify-content: space-around;
  justify-content: space-evenly;

  /* Align along CROSS axis (vertical if row) */
  align-items: stretch;     /* default: fill height */
  align-items: center;      /* vertically centered */
  align-items: flex-start;  /* top */
  align-items: flex-end;    /* bottom */

  /* Wrapping */
  flex-wrap: wrap;          /* wrap to next row */

  /* Gap between items */
  gap: 16px;
  gap: 10px 20px;          /* row-gap column-gap */
}

/* On children */
.item {
  flex: 1;                  /* grow and fill equally */
  flex: 0 0 200px;         /* don't grow, 200px wide */
  align-self: center;      /* override parent align-items */
  order: 2;                 /* reorder visually */
}
💡

Perfect centering trick: display: flex; justify-content: center; align-items: center; — this centers a child both horizontally and vertically inside the parent.

MODULE 10

Layout: CSS Grid

Two-dimensional layout — rows AND columns simultaneously

CSS Grid lets you define a grid of rows and columns and place items precisely anywhere within it. It's ideal for full page layouts, dashboards, and any 2D arrangement.

Grid Essentials
.grid {
  display: grid;

  /* Define columns */
  grid-template-columns: 200px 1fr 1fr;
  grid-template-columns: repeat(3, 1fr);       /* 3 equal columns */
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)); /* responsive! */

  /* Define rows */
  grid-template-rows: auto 1fr auto;    /* header content footer */

  /* Gap between cells */
  gap: 20px;
  column-gap: 24px;
  row-gap: 16px;

  /* Named areas (semantic layout) */
  grid-template-areas:
    "header header header"
    "sidebar main  main"
    "footer footer footer";
}

/* Assign items to named areas */
header { grid-area: header; }
aside  { grid-area: sidebar; }
main   { grid-area: main; }
footer { grid-area: footer; }

/* Manually span items */
.featured {
  grid-column: 1 / 3;      /* span from line 1 to 3 */
  grid-column: span 2;     /* span 2 columns */
  grid-row: 1 / 3;
}

Transitions & Animations

Motion in CSS — from simple hover effects to full keyframe animations

Transitions

Transitions animate a change from one CSS value to another — typically on :hover, :focus, or when a class is toggled.

CSS Transitions
.button {
  background: blue;
  transform: scale(1);

  /* property | duration | easing | delay */
  transition: all 0.3s ease 0s;

  /* Or specific properties */
  transition: background 0.3s ease, transform 0.2s ease;
}

.button:hover {
  background: darkblue;
  transform: scale(1.05);
}

/* Easing functions */
transition-timing-function: linear;
transition-timing-function: ease;       /* default */
transition-timing-function: ease-in;    /* slow start */
transition-timing-function: ease-out;   /* slow end */
transition-timing-function: ease-in-out;
transition-timing-function: cubic-bezier(0.4, 0, 0.2, 1); /* custom */

Keyframe Animations

@keyframes Animations
/* 1. Define the animation */
@keyframes slideIn {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes pulse {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.05); }
  100% { transform: scale(1); }
}

/* 2. Apply the animation */

  /* name | duration | easing | delay | iterations | direction */

.badge {

}

/* Pause on hover */
.badge:hover { animation-play-state: paused; }

/* Respect user's motion preferences */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; transition: none !important; }

MODULE 12

Responsive Design

One codebase, every screen size — mobile-first approach

Responsive design means your site looks good on every device — phones, tablets, and desktops. The key tools are media queries, fluid units, and flexible layouts.

Media Queries
/* Mobile-first: write base styles for mobile, then add breakpoints */

/* Base (mobile) — no query needed */
.grid {
  display: grid;
  grid-template-columns: 1fr;     /* Single column on mobile */
  gap: 16px;
}

/* Tablet — 768px and above */
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(2, 1fr); }
}

/* Desktop — 1024px and above */
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* Other conditions */
@media (max-width: 480px) { ... }     /* small phones only */
@media (orientation: landscape) { ... }
@media (hover: hover) { ... }         /* device has hover (non-touch) */
@media (prefers-color-scheme: dark) { ... } /* dark mode */

Control Flow

if/else, loops, switch — directing what your code does
Conditionals
// if / else if / else
if (score >= 90) {
  console.log('A grade');
} else if (score >= 70) {
  console.log('B grade');
} else {
  console.log('Needs improvement');
}

// Ternary operator — one-liner if/else
const label = score >= 60 ? 'Pass' : 'Fail';

// Switch
switch (day) {
  case 'Monday':
    console.log('Start of week');
    break;
  case 'Friday':
    console.log('Weekend soon!');
    break;
  default:
    console.log('Regular day');
}

// Nullish coalescing — use default if null/undefined
const username = user?.name ?? 'Guest';

// Optional chaining — safely access nested properties
const city = user?.address?.city; // undefined if any step is null
Loops
// for loop
for (let i = 0; i < 5; i++) {
  console.log(i); // 0, 1, 2, 3, 4
}

// while loop
let count = 0;
while (count < 3) {
  console.log(count++);
}

// for...of — iterate over arrays/strings
const fruits = ['apple', 'banana', 'mango'];
for (const fruit of fruits) {
  console.log(fruit);
}

// for...in — iterate over object keys
const person = { name: 'Alby', age: 25 };
for (const key in person) {
  console.log(key, person[key]);
}

// break & continue
for (let i = 0; i < 10; i++) {
  if (i === 5) break;    // stop loop
  if (i % 2 === 0) continue; // skip even
  console.log(i); // 1, 3
}

MODULE 16

Functions

Reusable blocks of code — the backbone of JavaScript
All Function Syntax Forms
// Function declaration — hoisted (usable before definition)
function add(a, b) {
  return a + b;
}

// Function expression — not hoisted
const multiply = function(a, b) {
  return a * b;
};

// Arrow function — concise, no own 'this'
const square = (n) => n * n;
const greet  = name => `Hello, ${name}!`;
const add2   = (a, b) => a + b;

// Arrow with block body — need explicit return
const process = (x) => {
  const doubled = x * 2;
  return doubled + 1;
};

// Default parameters
function createUser(name, role = 'member', active = true) {
  return { name, role, active };
}

// Rest parameters — collect remaining args
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
sum(1, 2, 3, 4); // 10

// Immediately Invoked Function Expression (IIFE)
(function() {
  // Runs immediately, private scope
})();

// Higher-order function — takes/returns functions
const withLogging = (fn) => (...args) => {
  console.log('Calling with:', args);
  return fn(...args);
};

Closures

A closure is when an inner function remembers variables from its outer function even after the outer function has finished.

Closures
function makeCounter(start = 0) {
  let count = start; // private variable

  return {
    increment: () => ++count,
    decrement: () => --count,
    value: () => count,
  };
}

const counter = makeCounter(10);
counter.increment(); // 11
counter.increment(); // 12
counter.value();     // 12

Array Creation & Access
const arr = [10, 20, 30, 40, 50];

arr[0]          // 10 — first element
arr[arr.length-1] // 50 — last element

Essential Array Methods
const nums = [1, 2, 3, 4, 5];

// ── Mutating methods (change original) ──
nums.push(6);       // Add to end → [1,2,3,4,5,6]
nums.pop();         // Remove from end → [1,2,3,4,5]
nums.unshift(0);   // Add to start → [0,1,2,3,4,5]
nums.shift();       // Remove from start → [1,2,3,4,5]
nums.splice(1, 2); // Remove 2 items at index 1
nums.sort((a,b) => a-b); // Sort ascending
nums.reverse();    // Reverse in place

// ── Non-mutating (return new array) ──

MODULE 24

How HTML, CSS & JS Connect

The full picture — the three languages working as one system

Understanding how these three layers interact is the most important concept in frontend development. They're not separate worlds — they're deeply entangled.

1. HTML is parsed

Browser reads HTML top to bottom. As it encounters elements, it builds the DOM tree in memory. This is the live object model JavaScript can access.

2. CSS is parsed

Browser loads CSS files (linked in <head>) and builds the CSSOM (CSS Object Model). Each selector is mapped to computed style values for matching elements.

3. Render Tree built

DOM + CSSOM = Render Tree. Only visible nodes are included (display:none excluded). Each node now has its computed styles.

4. Layout + Paint + Composite

Layout (reflow): browser calculates exact position and size of every element. Paint: fills in pixels. Composite: layers merged and drawn to screen.

5. JavaScript executes

JS runs after DOMContentLoaded (or is deferred). It can read and write to the DOM, change styles via CSSOM, and trigger layout/paint again. This is the live interaction loop.

JavaScript → DOM → CSS: Live Example

All Three Working Together

  .getElementById('theme-btn')
  .addEventListener('click', () => {
    document.body.classList.toggle('dark');
  });
// JS toggles a class → CSS does the visual work
// This is the clean separation pattern!

The Golden Rule: Let CSS handle visuals (transitions, styles). Use JavaScript only to toggle classes or update data. Never manually set 20 style properties in JS — that's CSS's job.

CSS Positioning

static, relative, absolute, fixed, sticky — and the z-index stacking system

The position property controls how an element is placed in the document flow. Combined with top/right/bottom/left, it lets you place elements precisely anywhere.

All Position Values
/* static (default) — follows normal document flow, top/left have no effect */
.box { position: static; }

/* relative — offset FROM its normal position (still takes up space) */
.nudged {
  position: relative;
  top: 10px;    /* moves DOWN 10px from where it would normally be */
  left: 20px;
}

/* absolute — removed from flow, positioned relative to nearest
   positioned ancestor (anything with position != static) */
.parent { position: relative; }  /* ← the anchor */
.child {
  position: absolute;
  top: 0; right: 0;  /* top-right corner of .parent */
}

/* fixed — relative to the VIEWPORT, stays in place on scroll */
.navbar {
  position: fixed;
  top: 0; left: 0; right: 0;
  z-index: 100;
}

/* sticky — normal flow UNTIL it hits a scroll threshold,
   then acts like fixed within its scroll container */
.sticky-header {
  position: sticky;
  top: 0;       /* sticks when it hits 0px from viewport top */
  z-index: 10;
}

Centering Patterns with Position

Classic Absolute Centering
/* Center absolute child in relative parent */
.overlay {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);  /* offset by half own size */
}

/* Full cover overlay */
.full-cover {
  position: absolute;
  inset: 0;  /* shorthand for top:0 right:0 bottom:0 left:0 */
  background: rgba(0,0,0,0.5);
}

z-index & Stacking Contexts

z-index Rules
/* z-index only works on positioned elements (non-static) */
.modal   { position: fixed;    z-index: 1000; }
.tooltip { position: absolute; z-index: 200; }
.navbar  { position: sticky;   z-index: 100; }

/* NEW stacking contexts are created by:
   - position + z-index (not auto)
   - transform (any value)
   - opacity < 1
   - filter, will-change, isolation: isolate
   Children can't escape their stacking context! */

/* isolation:isolate creates a stacking context without side effects */
.card { isolation: isolate; }

Import/export — the standard way to split code across files

Modules let you split your code into separate files and explicitly declare what each file exports and imports. Every file in a module system has its own scope — nothing leaks to global by default.

Scope

Global, Function & Block Scope
// Global scope — accessible everywhere
var globalVar = 'everywhere';
let globalLet = 'also global but not on window';

function outer() {
  // Function scope — var only visible inside this function
  var funcVar = 'function';
  let funcLet = 'also function';

  if (true) {
    // Block scope — let/const only visible in this block
    var blockVar = 'I leak to function scope!'; // ← var escapes blocks
    let blockLet = 'only inside this if block';
    const blockConst = 'same';
  }

  console.log(blockVar);    // ✅ accessible — var escapes the block
  console.log(blockLet);    // ❌ ReferenceError

  function inner() {
    // Scope chain: inner can see outer's variables
    console.log(funcLet);   // ✅ accessed via scope chain
    console.log(globalLet); // ✅ chain continues to global
  }
}

Hoisting

How Hoisting Works
// Function declarations are FULLY hoisted (callable before definition)
greet(); // ✅ Works! "Hello"
function greet() { console.log('Hello'); }

// var is hoisted but UNDEFINED until assigned
console.log(x); // undefined (not error)
var x = 5;
console.log(x); // 5

// let/const are hoisted but in "Temporal Dead Zone" (TDZ)
console.log(y); // ❌ ReferenceError: Cannot access before initialization
let y = 5;      // TDZ ends here

// Function expressions are NOT hoisted
double(3);       // ❌ TypeError: double is not a function
var double = n => n * 2;

Closures — Functions That Remember

Closure Patterns
// A closure captures variables from its enclosing scope
function createMultiplier(factor) {
  // 'factor' is captured by the returned function
  return (number) => number * factor;
}
const double = createMultiplier(2);
const triple = createMultiplier(3);
double(5); // 10
triple(5); // 15

// Real use: private state
function createStore(initialState) {
  let state = { ...initialState };
  const listeners = [];

  return {
    getState: () => ({ ...state }),
    setState(update) {
      state = { ...state, ...update };
      listeners.forEach(fn => fn(state));
    },
    subscribe(fn) { listeners.push(fn); }
  };
}

Prototype Chain & OOP

How inheritance really works in JavaScript — under the hood of class syntax

Every JavaScript object has an internal link to another object called its prototype. When you access a property that doesn't exist on the object, JS walks up the prototype chain until it finds it — or reaches null.

The Prototype Chain
const dog = { name: 'Rex', breed: 'Husky' };

// dog's prototype chain:
// dog → Object.prototype → null

// Object.prototype methods are available on all objects
dog.hasOwnProperty('name')    // true  — own property
dog.hasOwnProperty('toString') // false — inherited from Object.prototype

// Check/inspect the chain
Object.getPrototypeOf(dog) === Object.prototype  // true

// Create object with a specific prototype
const animal = { breathe() { return 'breathing'; } };
const cat = Object.create(animal); // cat's proto = animal
cat.name = 'Luna';
cat.breathe(); // ✅ 'breathing' — found on prototype

// 'in' checks own + prototype chain; hasOwnProperty checks own only
'breathe' in cat                     // true
cat.hasOwnProperty('breathe')       // false
Object.hasOwn(cat, 'breathe')       // false (modern version)
Constructor Functions vs Classes (Same Thing)
// ── Old way: Constructor function ──
function Person(name, age) {
  this.name = name;
  this.age = age;
}
Person.prototype.greet = function() {
  return `Hi, I'm ${this.name}`;
};

// ── New way: class syntax (syntactic sugar over above) ──
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  greet() { return `Hi, I'm ${this.name}`; }
}

const p = new Person('Alby', 25);
// p → Person.prototype → Object.prototype → null

// 'new' does 4 things:
// 1. Creates empty object {}
// 2. Sets its __proto__ to Constructor.prototype
// 3. Runs Constructor with 'this' = new object
// 4. Returns the new object

Generators & Iterators

Pausable functions, infinite sequences, and custom iteration

Generators are functions that can pause and resume. They're defined with function* and use yield to pause, returning a value to the caller each time.

Generator Basics
function* counter(start = 0) {
  while (true) {          // infinite loop is fine in generators!
    yield start++;         // pause here, hand back value
  }
}

const count = counter(1);   // creates generator object (doesn't run)
count.next(); // { value: 1, done: false }
count.next(); // { value: 2, done: false }
count.next(); // { value: 3, done: false }

// Finite generator
function* range(from, to, step = 1) {
  for (let i = from; i <= to; i += step) {
    yield i;
  }
}

// Generators are iterable — use for...of
for (const n of range(1, 10, 2)) {
  console.log(n); // 1, 3, 5, 7, 9
}

// Spread and destructure generators
[...range(1, 5)]            // [1, 2, 3, 4, 5]
const [a, b, c] = range(10, 30, 10); // a=10, b=20, c=30
yield* Delegation & Practical Patterns
// yield* delegates to another iterable
function* concat(...iterables) {
  for (const it of iterables) {
    yield* it;
  }
}
[...concat([1,2], [3,4], [5,6])] // [1,2,3,4,5,6]

// Passing values IN via next(value)
function* accumulate() {
  let total = 0;
  while (true) {
    const n = yield total;    // yield sends total, receives n
    if (n === null) return total;
    total += n;
  }
}
const acc = accumulate();
acc.next();    // start — { value: 0 }
acc.next(10); // { value: 10 }
acc.next(5);  // { value: 15 }

  }

    .forEach(t => { t.ariaSelected = 'false'; });

  tab.ariaSelected = 'true';
  document.getElementById(tab.getAttribute('aria-controls')).hidden = false;
});

JavaScript — The Core Logic

app.js — state & rendering
// State: single source of truth
let tasks = [
  { id: 1, text: 'Learn HTML structure',     done: true  },
  { id: 2, text: 'Style with CSS',             done: true  },
  { id: 3, text: 'Add interactivity with JS',  done: false },
  { id: 4, text: 'Build something awesome!',   done: false },
];

// Render: always derive DOM from state (never track DOM directly)
function render() {
  const list = document.getElementById('task-list');
  list.innerHTML = tasks.map(function(t) {
    return '<div class="item ' + (t.done ? 'done' : '') + '">'
      + '<div class="check" onclick="toggle(' + t.id + ')">'
      + (t.done ? '✓' : '') + '</div>'
      + '<span class="text">' + t.text + '</span>'
      + '<button class="del" onclick="remove(' + t.id + ')">✕</button>'
      + '</div>';
  }).join('');

  const remaining = tasks.filter(function(t) { return !t.done; }).length;
  document.getElementById('count').textContent =
    remaining + ' task' + (remaining !== 1 ? 's' : '') + ' remaining';
}

function toggle(id) {
  tasks = tasks.map(function(t) {
    return t.id === id
      ? Object.assign({}, t, { done: !t.done })  // immutable update
      : t;
  });
  render();
}

function addTask() {
  const input = document.getElementById('new-task');
  const text = input.value.trim();
  if (!text) return;
  tasks.push({ id: Date.now(), text, done: false });
  input.value = '';
  render();
}

// Enter key submits — keyboard accessibility
document.getElementById('new-task')
  .addEventListener('keydown', function(e) {
    if (e.key === 'Enter') addTask();
  });

render(); // initial render on page load
💡

Key pattern — state-driven rendering: Instead of manipulating individual DOM nodes, keep an array as the source of truth and call render() after every change. This is the same mental model used by React, Vue, and Svelte — just without the framework.
