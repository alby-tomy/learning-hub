How to use this

Step 1 — Project page first. Before touching the Q&A, read the Project Deep Dive page fully. You need to own that project description before answering any questions about it.

Step 2 — Easy questions, all topics. Filter by "Easy" and go through all 40. These form your base — get them solid.

Step 3 — Mark what you know. As you get comfortable with a question, mark it as "Known." This lets you focus remaining time on the ones that still feel shaky.

Step 4 — Medium → Hard. Work through Medium and Hard questions. These are the ones that actually differentiate you from other candidates.

Step 5 — Connect to your real work. Every answer should reference something you've actually built — LangChain, ChromaDB, Pytest, Jenkins, MCP. Interviewers trust specifics.
